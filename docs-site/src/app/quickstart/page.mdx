# Quick Start

Generate your first proof bundle in **60 seconds**.

## Installation

```bash
npm install -g @isl-lang/isl-verify
# or
npx @isl-lang/isl-verify
```

## Run Your First Scan

Navigate to your project directory and run:

```bash
npx isl-verify .
```

This will:
1. Scan all TypeScript/JavaScript files in your project
2. Run all available property provers
3. Generate a trust score (0-100)
4. Output a verdict: **VERIFIED**, **PARTIAL**, or **INSUFFICIENT**

## Generate a Proof Bundle

```bash
npx isl-verify . --proof-bundle
```

This creates `.isl-verify/proof-bundle.json` containing:
- **File hashes** (SHA-256) of every scanned file
- **Property proofs** with full evidence
- **HMAC-SHA256 signature** proving bundle integrity
- **Residual risks** - explicit list of what was NOT verified

## View the Report

```bash
# Pretty-print to terminal
npx isl-verify . --proof-bundle --format pretty

# Save as Markdown
npx isl-verify . --proof-bundle --format markdown > report.md

# GitHub PR comment format
npx isl-verify . --proof-bundle --format pr-comment > pr-comment.md
```

## Verify a Bundle

```bash
npx isl-verify verify-bundle .isl-verify/proof-bundle.json
```

This checks:
- âœ… Signature is valid (bundle hasn't been tampered with)
- âœ… File hashes match current code (code hasn't changed)
- âœ… Timestamp is recent (bundle isn't stale)

## Configure Properties

Create `.isl-verify.json` in your project root:

```json
{
  "provers": {
    "auth-coverage": {
      "enabled": true,
      "publicRoutes": ["/api/health", "/api/auth/*"]
    },
    "secret-exposure": {
      "enabled": true,
      "allowedPatterns": ["NEXT_PUBLIC_*"]
    }
  },
  "ci": {
    "failOn": ["FAILED"],
    "requiredProperties": ["import-integrity", "auth-coverage", "sql-injection"]
  }
}
```
